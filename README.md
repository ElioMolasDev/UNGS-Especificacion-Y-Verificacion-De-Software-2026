## Introduccion a la ingenieria de requerimientos 
### Proceso de software
Un **Proceso de software** es una serie de actividades relacionadas que conducen a la elaboracion de un producto de software. Existen distintos procesos, pero todos incluyen 4 actividades fundamentales:
* Especificaion: Donde se definen las funcionalidades del software como las restricciones.
* Diseño e implementación: Se desarrrolla el software para cumplir con la especificaion
* Validación: Se valida el software para que asegurarse que cumpla lo que el cliente quiere
* Evolución: El software debe evolucionar para satisfacer con las necesidades cambiantes del cliente
Las descripciones de los procesos deben incluir:
* Productos: Es el resultado de una actividad del proceso.
* Roles: Reflejan las responsabilidades de los usuarios que intervienen en el proceso
